┌───────────────────────────────────────────────────────────────────────────┐
│                            DATA FLOW EXAMPLE                               │
│                      (Creating a New Volume)                               │
├───────────────────────────────────────────────────────────────────────────┤
│                                                                            │
│  1. User fills form in Frontend (CreateVolumePage)                        │
│                    ↓                                                       │
│  2. Form validation (Zod schema)                                          │
│                    ↓                                                       │
│  3. POST /api/volumes (with JWT token)                                    │
│                    ↓                                                       │
│  4. Backend: Auth Middleware validates JWT                                │
│                    ↓                                                       │
│  5. Backend: Volume Controller receives request                           │
│                    ↓                                                       │
│  6. Backend: Volume Service validates & processes                         │
│                    ↓                                                       │
│  7. Backend: Calls Fusion Mock API (POST /v1/volumes)                     │
│                    ↓                                                       │
│  8. Fusion Mock: Creates volume with "creating" status                    │
│                    ↓                                                       │
│  9. Fusion Mock: Schedules status update to "ready" (5s)                  │
│                    ↓                                                       │
│ 10. Backend: Saves volume to SQLite database                              │
│                    ↓                                                       │
│ 11. Backend: Returns volume to Frontend                                   │
│                    ↓                                                       │
│ 12. Frontend: Displays success toast                                      │
│                    ↓                                                       │
│ 13. Frontend: Starts polling for status updates                           │
│                    ↓                                                       │
│ 14. After 5s: Status changes to "ready"                                   │
│                    ↓                                                       │
│ 15. Frontend: Shows "Volume ready" notification                           │
│                                                                            │
└───────────────────────────────────────────────────────────────────────────┘
